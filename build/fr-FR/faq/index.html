Foire Aux Questions

Généralités

Qu’est-ce que 1.1.1.1 ?

Lorsque vous demandez à visiter une application telle que cloudflare.com, votre ordinateur doit savoir sur quel serveur vous connecter pour pouvoir charger l'application. Les ordinateurs ne savent pas comment nommer pour adresser la traduction ; ils demandent donc à un serveur spécialisé de le faire pour eux..

Ce serveur spécialisé est appelé résolveur récursif DNS. La tâche du résolveur est de retrouver l’adresse liée à un nom donné, par exemple 2400:cb00:2048:1::c629:d7a2 pour cloudflare.com, et de la renvoyer à l’ordinateur qui a émis la demande.

Les ordinateurs sont configurés pour communiquer avec des résolveurs DNS spécifiques, identifiés par une adresse IP. En général, la configuration est gérée par votre fournisseur d’accès Internet (Orange ou Free, par exemple) si vous êtes chez vous ou sur un réseau Internet sans fil, et par votre administrateur réseau si vous êtes connecté à Internet au bureau. Vous pouvez également modifier le résolveur DNS configuré avec lequel votre ordinateur communique vous-même.

Que font les résolveurs DNS ?

Comment les résolveurs réussissent-ils à trouver l’adresse liée à un nom de domaine ? Ils travaillent vers l’arrière à partir du haut.

Chaque résolveur sait comment trouver le ’.’ invisible à la fin des noms de domaines (par exemple, cloudflare.com.). Il existe des centaines de serveurs partout dans le monde qui hébergent le fichier ’.’. Cloudflare héberge lui-même ce fichier sur tous ses serveurs à travers le monde au travers d’un partenariat avec l’ISC. Les résolveurs sont codés en dur pour connaître les adresses IP de ces serveurs.

Le résolveur demande à l'un des serveurs racine où trouver le prochain lien dans la chaîne, le TLD ou la fin du domaine. Par exemple, .com et .org sont des TLD. Heureusement, les serveurs racine stockent les emplacements de tous les serveurs TLD, afin de pouvoir indiquer quelle adresse IP le résolveur DNS doit demander ensuite.

Le résolveur demande ensuite aux serveurs TLD où il peut trouver le domaine qu’il recherche. Par exemple, un résolveur pourrait demander à .com où trouver cloudflare.com. Le TLD héberge un fichier contenant l’emplacement de chaque domaine utilisant ce TLD. Une fois que le résolveur a l’adresse IP finale, il renvoie la réponse à l’ordinateur qui l’a demandée.

L’ensemble du système, comprenant les serveurs qui hébergent l’information (DNS autoritaires) et ceux qui recherchent l’information (résolveurs DNS), est appelé Système de noms de domaine (DNS). Le DNS est comme Google Maps pour Internet. Il traduit le nom des emplacements en adresses afin de vous aider à les atteindre.

Est-ce que 1.1.1.1 est compatible avec IPv6 ?

Oui, 1.1.1.1. est entièrement compatible avec IPv6.

Comment puis-je effacer le cache DNS de 1.1.1.1 ?

Vous pouvez actualiser le cache DNS de 1.1.1.1 pour les noms de domaines en utilisant l’outil de purge du cache. Vous devez saisir le nom de domaine, choisir le(s) type(s) d’enregistrement DNS et cliquer sur le bouton « Purger le cache ».

Est-ce que 1.1.1.1 fonctionne en Chine ?

Pas vraiment. Vous pouvez configurer votre DNS pour qu’il pointe vers 1.1.1.1 mais il n’atteindra pas l’un de nos centres de données situés en Chine. La demande sera plutôt redirigé vers l'un de nos centres de données en dehors de la Chine continentale.

Application mobile

Comment fonctionne l’application mobile ?

L'application exécute un résolveur local dans votre téléphone mobile qui demande à toutes les autres applications d'envoyer leurs demandes DNS à l'adresse de ce résolveur. Une fois que ces requêtes DNS atteignent le résolveur local, celui-ci les chiffre et les envoie à 1.1.1.1.

L’application 1.1.1.1 est-elle un VPN ?

Non. Une application VPN fait généralement passer tout le trafic de votre réseau par un proxy. L’application 1.1.1.1 passe seulement outre pour sécuriser le trafic DNS de votre téléphone mobile.

Que sont les journaux DNS ?

Les journaux DNS sur iOS contiennent toutes les requêtes DNS émises par votre téléphone. Vos applications et votre navigateur mobile génèrent ces requêtes DNS.

À quoi servent les journaux de console ?

Les journaux de console permettent à Cloudflare de déboguer votre connexion.

Comment puis-je proposer une amélioration ?

Nous lisons toutes vos suggestions. Vous pouvez secouer votre téléphone lorsque vous êtes dans l’application pour envoyer vos propositions d’améliorations. À défaut, allez dans le menu de l’application et appuyez sur « Aide » pour envoyer votre commentaire.

Quelle est la différence entre l’utilisation de DNS sur TLS et de DNS sur HTTPS ?

Ces deux options chiffrent les requêtes DNS simples à partir du téléphone.

DNS sur HTTPS utilise le port 443 et DNS sur TLS le port 853. Sur certains réseaux, l’un de ces ports peut être bloqué. Si le port 443 est bloqué, utilisez DNS sur TLS. Si le port 853 est bloqué, utilisez DNS sur HTTPS. Dans certains cas, DNS sur TLS est plus rapide que DNS sur HTTPS, et vice-versa.

Comment puis-je envoyer un rapport de bogue ?

Vous pouvez secouer l’application pour envoyer un rapport de bogue. En faisant cela, l’application joint une capture d’écran et les journaux de console au rapport de bogue. Le journal de console nous aide à déboguer votre connexion et à améliorer votre expérience générale sur l’application.

Mon Internet a cessé de fonctionner lorsque j’utilisais l’application 1.1.1.1. Que dois-je faire ?

Si cela se produit, essayez de désactiver l'application et de la réactiver à l'aide de l’option de basculement de la page d'accueil de l'application. Si le problème persiste, veuillez désactiver l’application et nous envoyer un rapport de bogue.

L’application 1.1.1.1 est bloquée sur « CONNEXION » ou « RÉAFFIRMATION ». Que dois-je faire ?

Cela peut arriver lorsque votre connexion réseau est temporairement interrompue ou si votre réseau de données bloque les connexions 1.1.1.1 :

  • Désactivez l’application en utilisant l’option de basculement de la page d’accueil.
  • Ouvrez l’application Paramètres iOS et accédez à Paramètres > Général > Réinitialiser.
  • Appuyez sur Réinitialiser les paramètres réseau.
  • Redémarrez votre appareil ; assurez-vous d’être connecté au Wi-Fi.
  • Essayez de relancer l’application.

Si la connexion n’est pas autorisée, désactivez l’application 1.1.1.1 et envoyez-nous un rapport de bogue.

Je ne peux pas cliquer sur OK dans la boîte de dialogue de l’installation du VPN. Que dois-je faire ?

Si vous ne pouvez pas cliquer sur OK, il se peut qu'il y ait une autre application en haut de la boîte de dialogue. Certaines applications connues pour causer ce problème sont : Lux Brightness, Night Mode, Twilight, FileManager. Pour éviter ce problème, fermez ou désinstallez toutes les applications à risque.

J’ai laissé l’application active mais elle s’est éteinte après un moment. Que s’est-il passé ?

Votre téléphone tente de gérer la batterie en désactivant l’application. Par exemple, la plupart des appareils Huawei sont connus pour leur gestion agressive de la mémoire et de l’alimentation.

Vous pouvez régler ce problème en suivant les étapes ci-dessous :

  • Allez dans les « Réglages » sur votre téléphone.
  • Sélectionnez « Avancé »
  • Visitez « Gestion de la batterie »
  • Ouvrez « Applications protégées »
  • Vous pouvez sélectionner « Autoriser les applications à continuer à fonctionner après la désactivation de l'écran » pour l’application 1.1.1.1.

Pourquoi existe-t-il une notification persistante pour l’application 1.1.1.1 ?

Si vous utilisez Android 8 ou une version supérieure, vous verrez une notification persistante depuis l’application 1.1.1.1 indiquant que « vos requêtes DNS sont privées et plus rapides ».

Elle est requise par le système Android (8.0+) pour toutes les applications ayant une activité en arrière-plan.

Nous définissons la priorité de la notification sur « Faible » pour qu’elle n’interfère pas avec d’autres notifications importante de votre téléphone.

Vous pouvez masquer la notification comme ceci. Sachez que cela vous empêchera de recevoir d’autres notifications de l’application 1.1.1.1. Procédez de la manière suivante :

  1. Accédez à vos Réglages.
  2. Applications et notifications.
  3. Ouvrez le réglage de l’application 1.1.1.1 dans la liste des applications.
  4. Tapez sur « Notifications » et désactivez-les.

J’ai désactivé les notifications pour l’application 1.1.1.1 mais je vois maintenant une autre notification selon laquelle « 1.1.1.1 utilise la batterie ». Dois-je m’inquiéter ? Comment la retirer ?

Ne vous inquiétez pas. Étant donné que vous avez désactivé la notification pour l’application, le système d’exploitation Android envoie maintenant une autre notification vous informant de l’utilisation de la batterie, non pas parce qu’elle consomme beaucoup la batterie, mais parce qu’elle s’exécute en arrière plan (dans ce cas, toutes vos requêtes DNS sont sécurisées !).

Voici comment vous pouvez la retirer :

  1. Appuyez longuement sur la notification jusqu’à ce qu’elle change et affiche une option de basculement.
  2. Cliquez sur cette option pour la griser.
  3. Sélectionnez « Effectué » pour désactiver la notification.

Je ne peux pas ajouter de profil VPN. Comment y remédier ?

Vous utilisez probablement un autre VPN pour lequel le paramètre « Toujours activé » est activé. Veuillez le désactiver et activer 1.1.1.1.
