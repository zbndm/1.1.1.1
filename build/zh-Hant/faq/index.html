常見問題集

一般

什麼是 1.1.1.1？

當您要求瀏覽 cloudflare.com 之類的應用程式時，您的電腦需要知道要連線到哪一台伺服器，才能載入應用程式。電腦並不知道如何將名稱轉譯成位址，因此會請一台專門的伺服器幫忙轉譯。

此專門的伺服器稱為 DNS 遞迴解析服務。此解析服務的職責是尋找指定名稱的實際位址 (像 cloudflare.com 就是 2400:cb00:2048:1::c629:d7a2)，然後將其傳回詢問這項資訊的電腦。

電腦已設定成與特定 DNS 解析服務 (以 IP 位址識別) 聯繫。一般說來，如果是透過家用或無線網路連上網際網路，此設定是由 ISP (如 Comcast 或 AT&T) 所管理；如果透過公司網路連上網際網路，此設定是由公司的網路系統管理員所管理。您亦可自行變更要讓電腦聯繫的 DNS 解析服務。

DNS 解析服務能做些什麼？

解析服務怎麼知道要如何找到網域名稱的位址？畢竟其是要從頂層倒推回去。

每個解析服務都知道如何找到網域名稱結尾隱藏的「.」(如 cloudflare.com.)。全球有數以百計的伺服器裝載了「.」檔案。拜與 ISC 合作之賜，Cloudflare 位於世界各地的伺服器全都裝載了該檔案。解析服務經硬式編碼，知道這些伺服器的 IP 位址。

解析服務會詢問其中一台根伺服器要到哪裡找到連結鏈中的下一個連結，也就是頂層網域或網域後綴。頂層網域的範例像是 .com 或 .org。幸運的是，根伺服器會存放所有頂層網域伺服器的位置，因此可以傳回 DNS 解析服務接下來應詢問的 IP 位址。

然後，解析伺服器便會向頂層網域的伺服器詢問哪裡可以找到所要找的網域，例如，解析服務可能會向 .com 詢問哪裡可以找到 cloudflare.com。頂層網域裝載了一個檔案，當中包含所有使用該頂層網域的網域之位置。解析服務在取得最終 IP 位置後，就會將答案傳回查詢的電腦。

裝載這些資訊 (稱為權威 DNS) 及搜尋這些資訊 (DNS 解析服務) 的這整個伺服器系統稱為網域名稱系統 (DNS)。DNS 好比網際網路上的 Google 地圖，將地名轉譯為地址，讓您能夠思考該如何前往目的地。

1.1.1.1 是否具備 IPv6 支援？

是，1.1.1.1 完全支援 IPv6。

如何清除 1.1.1.1 的 DNS 快取？

您可以使用清除快取工具來重新整理 1.1.1.1 的 DNS 網域名稱快取。您必須輸入網域名稱、選取 DNS 記錄類型，然後點擊 [清除快取] 按鈕。

1.1.1.1 在中國境內是否可使用？

不盡然。您可以將 DNS 設為指向 1.1.1.1，但後者無法聯繫上我們在中國境內的資料中心。要求會改被路由傳送至我們在中國大陸境外的資料中心。

行動裝置 App

行動裝置 App 如何運作？

行動裝置 App 會在您的行動電話內執行本機解析服務，由其請所有其他 App 將 DNS 要求傳送至此解析服務的位置。待這些 DNS 要求送達本機解析服務後，本機解析服務便會先將這些內容加密，然後再傳送至 1.1.1.1。

1.1.1.1 App 是一種 VPN 嗎？

不是。VPN App 通常會代理您所有的網路流量。1.1.1.1 App 只會覆寫和保護您行動電話的 DNS 流量。

什麼是 DNS 記錄？

iOS 中的 DNS 記錄包含您手機上發出的所有 DNS 查詢。這些 DNS 查詢通常是由您的 App 和行動瀏覽器所產生。

主控台記錄是做什麼用的？

主控台記錄的存在是要協助 Cloudflare 對您的連線進行偵錯。

如何提出改進建議？

您所有的建議我們都會看過。您可以在此 App 中搖動手機，然後就能傳送改進建議。或者，到此 App 的選單中點一下 [說明] 後，同樣能夠傳送您的意見。

使用 DNS over TLS 與 DNS over HTTPS 有什麼區別？

DNS over TLS 與 DNS over HTTPS 都會將手機上發出的一般 DNS 查詢進行加密。

DNS over HTTPS 使用連接埠 443，而 DNS over TLS 使用連接埠 853。在某些網路中，上述其中一個連接埠可能會遭封鎖。如果連接埠 443 遭封鎖，則應使用 DNS over TLS。如果連接埠 853 遭封鎖，則應使用 DNS over HTTPS。DNS over TLS 在某些情況下的速度可能會比 DNS over HTTPS 還要快，在其他情況下則可能相反。

如何傳送錯誤報告？

您可以搖動此 App，然後就能傳送錯誤報告。當您這麼做時，此 App 會隨錯誤報告附加螢幕擷取畫面和主控台記錄。主控台記錄可協助我們偵錯您的連線，改善您的 App 整體使用體驗。

網際網路連線在我使用 1.1.1.1 App 時中斷。我該怎麼辦？

如果發生此情況，請試試利用此 App 首頁的切換開關，將此 App 關閉再開啟。如果此問題依然存在，請停用此 App 並向我們傳送錯誤報告。

1.1.1.1 App 在「正在連線」或「正在恢復」時卡住。我該怎麼辦？

如果您的網路連線臨時中斷，或是您的數據連線封鎖了 1.1.1.1 連線，就可能會發生此情況。

  • 請使用此 App 首頁的切換開關關閉此 App。
  • 開啟 iOS 的 [設定] App，巡覽至 [設定] > [一般] > [重設]。
  • 按一下 [重設網路設定]。
  • 重新啟動裝置；確定 Wifi 已連線。
  • 重試讓此 App 連線。

如果連線仍未恢復，請停用 1.1.1.1 App 並向我們傳送錯誤報告。

VPN 安裝對話方塊中的 [確定] 按不了。我該怎麼辦？

如果 [確定] 按鈕按不了，可能有其他應用程式蓋在對話方塊上。已知可能會造成此問題的一些 App 有 Lux Brightness、Night Mode、Twilight、FileManager 等等。若要避免此問題，請關閉或解除安裝所有可能會造成此問題的 App。

我啟用了此 App，但它沒過一會就關閉了。發生什麼事了？

您的手機正為了管理電池耗電量，而嘗試停用此 App。例如，大部份華為裝置均以其激進的記憶體與電源管理功能而著稱。

您可以透過以下步驟來修復此問題：

  • 前往手機的 [設定]。
  • 選取 [進階]
  • 進入 [電池管理系統]
  • 開啟 [受保護 App]
  • 您可以對 1.1.1.1 App 選取 [螢幕關閉後，允許 App 繼續運作]。

為什麼 1.1.1.1 App 有則通知一直不消失？

如果您使用的是 Android 8 或更高版本，您將看到有則表示「您的 DNS 查詢是私密的且更快」的 1.1.1.1 App 通知一直不消失。

對於所有在背景有活動的 App，Android OS (8.0+) 都要求這樣做。

我們將該則通知的優先順序設定為 [低]，以免干擾您注意手機上的其他重要通知。

您可以執行以下步驟來隱藏通知。請注意，您將因此無法接收 1.1.1.1 App 的其他通知。步驟如下：

  1. 前往手機上的 [設定]。
  2. [應用程式與通知]。
  3. 從 App 清單中開啟 1.1.1.1 App 設定。
  4. 點一下 [通知] 然後將其停用。

我停用了 1.1.1.1 App 的通知，但現在我看到另一則通知，表示「1.1.1.1 正在耗用電池」。我應該擔心嗎？如何將此通知移除？

您無需擔心。由於您已停用此 App 的通知，Android 作業系統現在會傳送另一則通知，警告您電量電池正在耗用中，但這並不代表此 App 耗用了大量電池電量，而是代表此 App 正在背景執行（也就是說，它正在保護您所有的 DNS 查詢！）。

以下是移除此通知的方法：

  1. 長按通知直到通知發生變化並顯示切換開關。
  2. 點一下切換開關，確定其呈現灰色停用的狀態。
  3. 選取 [完成] 以停用通知。

我新增不了 VPN 設定檔。該如何解決？

您可能正在使用另一個啟用了 [始終開啟] 設定的 VPN。請先停用該 VPN，然後再啟用 1.1.1.1。
