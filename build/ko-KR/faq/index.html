FAQ(질문과 대답)

일반

1.1.1.1은 무엇입니까?

사용자가 cloudflare.com 같은 애플리케이션 방문을 요청할 때 컴퓨터가 애플리케이션을 로드하려면 사용자를 어느 서버에 연결해야 하는지를 알아야 합니다. 컴퓨터는 이 이름-주소 변환을 수행하는 방법을 모르므로 특수화된 서버에 작업을 대신 요청합니다.

이 특수화된 서버를 DNS 재귀 확인자라고 합니다. 이 확인자의 작업은 지정된 이름의 주소(예: cloudflare.com의 경우 2400:cb00:2048:1::c629:d7a2)를 찾고, 요청한 컴퓨터에 이 주소를 반환하는 것입니다.

컴퓨터는 IP 주소로 식별되는 특정 DNS 확인자와 통신하도록 구성됩니다. 일반적으로 이 구성은 집 또는 무선 인터넷을 사용하는 경우 ISP(예: Comcast 또는 AT&T)에서 관리하고, 회사 인터넷에 연결된 경우 네트워크 관리자가 관리합니다. 컴퓨터와 사용자 간의 통신에 구성된 DNS 확인자를 변경할 수도 있습니다.

DNS 확인자가 수행하는 작업은 무엇입니까?

확인자는 도메인 이름의 주소를 어떻게 찾습니까? 확인자는 하향식으로 작동합니다.

모든 확인자는 도메인 이름의 끝에서 보이지 않는 ‘.’를 찾을 수 있습니다(예: cloudflare.com.). 전 세계에는 ‘.’ 파일을 호스트하는 수백 개의 서버가 있습니다. Cloudflare는 ISC와의 파트너십을 통해 전 세계의 모든 Cloudflare 서버에서 이 파일을 직접 호스트합니다. 확인자는 이러한 서버의 IP 주소를 알 수 있도록 하드코딩되어 있습니다.

확인자는 루트 서버 중 하나에 체인, TLD 또는 도메인 끝에서 다음 링크가 있는 위치를 묻습니다. TLD의 예로는 .com 또는 .org가 있습니다. 다행히 루트 서버에는 모든 TLD 서버의 위치가 저장되므로 루트 서버는 DNS 확인자가 다음으로 물어야 하는 IP 주소를 반환할 수 있습니다.

다음으로 확인자는 TLD의 서버에 조회 중인 도메인의 위치를 묻습니다. 예를 들어 확인자는 .com에 cloudflare.com의 위치를 물을 수 있습니다. TLD는 TLD를 사용하는 모든 도메인의 위치를 포함하는 파일을 호스트합니다. 확인자는 최종 IP 주소를 받은 후 요청한 컴퓨터로 응답을 반환합니다.

정보를 호스트하는 서버(권한 있는 DNS)와 정보를 구하는 서버(DNS 확인자)가 포함되는 이 전체 시스템을 DNS(Domain Name System)라고 합니다. DNS는 인터넷의 Google Maps와 같습니다. 장소의 이름을 주소로 변환하여 경로를 알려줍니다.

1.1.1.1은 IPv6을 지원합니까?

예. 1.1.1.1은 IPv6을 완벽하게 지원합니다.

1.1.1.1의 DNS 캐시를 지우려면 어떻게 해야 합니까?

캐시 제거 도구를 사용하여 도메인 이름에 대한 1.1.1.1의 DNS 캐시를 새로 고칠 수 있습니다. 도메인 이름을 입력하고 DNS 레코드 유형을 선택한 후 ‘캐시 제거’ 버튼을 누르면 됩니다.

1.1.1.1은 중국 내에서 작동합니까?

아닙니다. 1.1.1.1을 가리키도록 DNS를 구성할 수는 있지만 중국 내의 데이터 센터에 연결되지 않습니다. 대신 요청이 중국 본토 외부의 데이터 센터로 라우트됩니다.

모바일 앱

모바일 앱은 어떻게 작동합니까?

모바일 앱은 휴대폰 안에서 다른 모든 앱에 DNS 요청을 이 확인자의 주소로 전송할 것을 요청하는 로컬 확인자를 실행합니다. 이러한 DNS 요청이 로컬 확인자에 전송되면 로컬 확인자가 요청을 암호화하고 1.1.1.1로 전송합니다.

1.1.1.1 앱은 VPN입니까?

아니요. VPN 앱은 일반적으로 모든 네트워크 트래픽을 프록싱합니다. 1.1.1.1 앱은 휴대폰의 DNS 트래픽만 재정의하고 보호합니다.

DNS 로그는 무엇입니까?

iOS의 DNS 로그에는 휴대폰에서 생성된 모든 DNS 쿼리가 포함됩니다. 일반적으로 이러한 DNS 쿼리는 앱 및 모바일 브라우저에서 생성됩니다.

콘솔 로그의 용도는 무엇입니까?

콘솔 로그는 Cloudflare에서 사용자 연결을 디버그하는 데 사용됩니다.

개선 사항을 제안하려면 어떻게 해야 합니까?

당사는 사용자의 제안을 모두 읽습니다. 앱 안에 있을 때 휴대폰을 흔들어 개선 제안을 보낼 수 있습니다. 또는 앱 메뉴로 이동하고 ‘도움말’을 눌러 피드백을 보낼 수 있습니다.

DNS over TLS와 DNS over HTTPS를 사용할 때의 차이점은 무엇입니까?

DNS over TLS와 DNS over HTTPS 모두 휴대폰의 일반 DNS 쿼리를 암호화합니다.

DNS over HTTPS는 포트 443을 사용하고 DNS over TLS는 포트 853을 사용합니다. 일부 네트워크에서는 이러한 포트 중 하나가 차단될 수 있습니다. 포트 443이 차단된 경우 DNS over TLS를 사용해야 합니다. 포트 853이 차단된 경우에는 DNS over HTTPS를 사용해야 합니다. 경우에 따라 DNS over TLS가 DNS over HTTPS보다 빠르거나 그 반대일 수 있습니다.

버그 보고서를 보내려면 어떻게 해야 합니까?

앱을 흔들어 버그 보고서를 보낼 수 있습니다. 이렇게 하면 앱이 스크린샷과 콘솔 로그를 버그 보고서와 함께 첨부합니다. 콘솔 로그는 연결을 디버그하고 전체 앱 경험을 개선하는 데 도움이 됩니다.

1.1.1.1 앱을 사용하는 동안 인터넷이 작동을 멈췄습니다. 어떻게 해야 합니까?

이 문제가 발생하면 앱 홈 페이지의 토글을 사용하여 앱을 끈 후 다시 켜십시오. 문제가 지속되면 앱을 비활성화하고 버그 보고서를 보내 주십시오.

1.1.1.1 앱이 ‘CONNECTING’ 또는 ‘REASSERTING’ 상태에서 중단되었습니다. 어떻게 해야 합니까?

네트워크 연결이 일시적으로 중단되었거나 데이터 연결이 1.1.1.1 연결을 차단하는 경우 이 문제가 발생할 수 있습니다.

  • 앱 홈 페이지의 토글을 사용하여 앱을 비활성화합니다.
  • iOS 설정 앱을 열고 설정 > 일반 > 재설정으로 이동합니다.
  • 네트워크 설정 재설정을 누릅니다.
  • 기기를 재부팅하고 Wifi가 연결되는지 확인합니다.
  • 앱을 다시 연결해 보십시오.

연결이 허용되지 않으면 1.1.1.1 앱을 비활성화하고 당사로 버그 보고서를 보내 주십시오.

VPN 설치 대화 상자에서 확인을 클릭할 수 없습니다. 어떻게 해야 합니까?

확인 버튼을 클릭할 수 없다면 대화 상자 위에 다른 애플리케이션이 있는 문제일 수 있습니다. Lux Brightness, Night Mode, Twilight, FileManager 같은 일부 앱이 이러한 문제를 일으키는 것으로 알려져 있습니다. 이 문제를 방지하려면 이 문제를 야기할 수 있는 모든 앱을 닫거나 제거하십시오.

앱을 활성화한 상태로 유지했는데 잠시 후 꺼진 것을 알았습니다. 어떻게 된 일입니까?

휴대폰이 배터리 관리를 위해 앱을 비활성화한 것입니다. 예를 들어 대부분의 Huawei 기기는 메모리 및 전원을 적극적으로 관리하는 것으로 알려져 있습니다.

이 문제는 아래의 단계를 수행하여 해결할 수 있습니다.

  • 휴대폰의 ‘설정’으로 이동합니다.
  • ‘고급’을 선택합니다.
  • ‘배터리 관리자’로 이동합니다.
  • ‘보호되는 앱’을 엽니다.
  • 1.1.1.1 앱에 대해 “화면이 꺼진 후 계속해서 앱 실행”을 선택할 수 있습니다.

1.1.1.1 앱에 대한 영구 알림이 표시되는 이유는 무엇입니까?

Android 8 이상을 사용하는 경우 1.1.1.1 앱에서 “DNS 쿼리가 비공개이고 더 빨라짐”이라는 영구 알림이 표시됩니다.

이 알림은 Android OS(8.0+)에서 백그라운드 작업을 실행하는 모든 앱에 필수적으로 표시됩니다.

휴대폰의 다른 중요한 알림을 방해하지 않도록 알림의 우선 순위를 “낮음”으로 설정했습니다.

이 알림을 숨기는 방법이 한 가지 있습니다. 하지만 이렇게 하면 1.1.1.1 앱에서 다른 알림이 수신되지 않습니다. 알림을 숨기려면 다음과 같이 하십시오.

  1. “설정”으로 이동합니다.
  2. “앱 및 알림”으로 이동합니다.
  3. 앱 목록에서 1.1.1.1 앱 설정을 엽니다.
  4. “알림”을 누르고 비활성화합니다.

1.1.1.1 앱의 알림을 비활성화했지만 “1.1.1.1이 배터리 사용 중”이라는 다른 알림이 표시됩니다. 문제가 됩니까? 이 알림은 어떻게 없앱니까?

걱정하지 않아도 됩니다. 앱의 알림을 비활성화했기 때문에 이제 Android OS가 배터리 사용량에 대해 경고하는 다른 알림을 보내는 것입니다. 단, 이 알림은 앱이 배터리를 많이 소모하기 때문이 아니라 백그라운드로 실행되고 있기 때문에 표시되는 것입니다(이 경우 모든 DNS 쿼리에 보안 적용).

이 알림을 없애는 방법은 다음과 같습니다.

  1. 알림이 바뀌고 토글이 표시될 때까지 알림을 길게 누릅니다.
  2. 토글을 눌러 회색으로 비활성화되게 합니다.
  3. “완료”를 선택하여 알림을 비활성화합니다.

VPN 프로필을 추가할 수 없습니다. 이 문제는 어떻게 해결합니까?

“상시” 설정이 활성화된 다른 VPN을 사용 중인 것 같습니다. 해당 VPN을 비활성화하고 1.1.1.1을 활성화하십시오.
