Pertanyaan yang Sering Diajukan

Umum

Apa itu 1.1.1.1?

Saat Anda meminta untuk mengunjungi aplikasi seperti cloudflare.com, komputer Anda perlu mengetahui server mana yang menyambungkan Anda agar dapat memuat aplikasi. Komputer tidak tahu cara menerjemahkan nama untuk mendapatkan alamat, jadi komputer meminta server khusus untuk melakukannya.

Server khusus ini disebut resolver rekursif DNS. Tugas resolver adalah menemukan alamat untuk nama tertentu, seperti 2400:cb00:2048:1::c629:d7a2 untuk cloudflare.com, dan mengembalikannya ke komputer yang memintanya.

Komputer dikonfigurasikan untuk berbicara dengan resolver DNS khusus, yang diidentifikasi oleh alamat IP. Biasanya, konfigurasi dikelola oleh ISP Anda (seperti Comcast atau AT&T) jika Anda berada di Internet rumah atau nirkabel, dan oleh administrator jaringan Anda jika tersambung ke Internet kantor. Anda juga dapat mengubah resolver DNS yang dikonfigurasi komputer untuk berbicara kepada Anda sendiri.

Apa yang dilakukan resolver DNS?

Bagaimana resolver mengetahui cara menemukan alamat nama domain? Resolver bekerja mundur dari atas.

Setiap resolver mengetahui cara menemukan ‘.’ yang tidak tampak pada akhir nama domain (mis. cloudfire.com). Ada ratusan server di seluruh dunia yang menghosting file ‘.’. Cloudflare sendiri menghosting file tersebut pada semua servernya sendiri di seluruh dunia melalui kemitraan dengan ISC. Resolver di-kode keras untuk mengetahui alamat IP server tersebut.

Resolver meminta salah satu server root tempat menemukan tautan berikutnya di rantai, TLD atau akhiran domain. Contoh TLD adalah .com atau .org. Untungnya server root menyimpan lokasi dari semua server TLD, sehingga server dapat mengembalikan alamat IP mana yang selanjutnya harus ditanya oleh resolver DNS.

Resolver kemudian menanyakan server TLD tempat ia dapat menemukan domain yang dicari, misalnya, resolver mungkin bertanya ke .com di mana menemukan cloudflare.com. TLD menghosting file yang berisi lokasi setiap domain yang menggunakan TLD. Setelah resolver memiliki alamat IP final, resolver mengembalikan jawaban ke komputer yang menanyakannya.

Seluruh sistem ini - dengan server yang menghosting informasi (disebut DNS otoritatif) dan server yang mencari informasi (resolver DNS) disebut Sistem Nama Domain (Domain Name System/DNS). DNS itu seperti Google Maps untuk Internet. DNS menerjemahkan nama tempat menjadi alamat sehingga Anda dapat mengetahui jalan menuju ke sana.

Apakah 1.1.1.1 memiliki dukungan IPv6?

Ya, 1.1.1.1 memiliki dukungan IPv6 penuh.

Bagaimana cara saya menghapus cache DNS 1.1.1.1?

Anda dapat menyegarkan cache DNS 1.1.1.1 untuk nama domain dengan menggunakan alat pembersih cache. Anda harus memasukkan nama domain, pilih tipe (atau tipe-tipe) rekaman DNS dan tekan tombol ‘Bersihkan Cache’.

Apakah 1.1.1.1 dapat digunakan di dalam Tiongkok?

Tidak juga. Anda dapat mengonfigurasi DNS Anda untuk mengarah ke 1.1.1.1 tetapi tidak akan mencapai salah satu pusat data kami di dalam Tiongkok. Sebaliknya, permintaan akan mengarahkan ke salah satu pusat data kami di luar Daratan Tiongkok.

Aplikasi Seluler

Bagaimana cara kerja aplikasi seluler?

Aplikasi ini menjalankan resolver lokal di dalam ponsel Anda yang meminta semua aplikasi lain untuk mengirim permintaan DNS-nya ke alamat resolver ini. Setelah permintaan DNS ini mencapai resolver lokal, DNS mengenkripsi dan mengirimnya ke 1.1.1.1.

Apakah aplikasi 1.1.1.1 adalah VPN?

Bukan. Aplikasi VPN biasanya memberi proksi semua lalu-lintas jaringan Anda. Aplikasi 1.1.1.1 hanya menimpa dan mengamankan lalu-lintas DNS ponsel Anda.

Apa itu log DNS?

Log DNS pada iOS berisi semua kueri DNS yang dibuat ponsel Anda. Aplikasi dan browser ponsel Anda biasanya menghasilkan kueri DNS ini.

Apa fungsi log konsol?

Log konsol ada untuk membantu Cloudfire melakukan debug pada sambungan Anda.

Bagaimana cara saya menyarankan peningkatan?

Kami membaca semua saran Anda. Anda dapat menggoyangkan ponsel saat dalam aplikasi dan mengirim saran peningkatan Anda. Atau, Anda dapat masuk ke menu aplikasi dan ketuk ‘Bantuan’ untuk mengirimkan masukan.

Apa perbedaan antara menggunakan DNS melalui TLS dan DNS melalui HTTPS?

Baik DNS melalui TLS dan DNS melalui HTTPS mengenkripsi kueri DNS biasa dari ponsel.

DNS melalui HTTPS menggunakan port 443 dan DNS melalui TLS menggunakan port 853. Di beberapa jaringan, salah satu port ini mungkin diblokir. Jika port 443 diblokir, Anda harus menggunakan DNS melalui TLS. Jika port 853 diblokir, Anda harus menggunakan DNS melalui HTTPS. Dalam beberapa kasus, DNS melalui TLS mungkin lebih cepat dibandingkan DNS melalui HTTPS atau sebaliknya.

Bagaimana saya dapat mengirimkan laporan bug?

Anda dapat menggoyangkan aplikasi dan mengirimkan laporan bug. Saat Anda melakukannya, aplikasi melampirkan tangkapan layar dan log konsol bersama dengan laporan bug. Log konsol membantu kami melakukan debug pada sambungan Anda dan meningkatkan keseluruhan pengalaman aplikasi Anda.

Internet saya berhenti bekerja saat saya sedang menggunakan aplikasi 1.1.1.1. Apa yang harus saya lakukan?

Jika ini terjadi, coba matikan aplikasi dan nyalakan kembali dengan menggunakan toggle dari halaman beranda aplikasi. Jika masalah tetap ada, nonaktifkan aplikasi dan kirimkan laporan bug kepada kami.

Aplikasi 1.1.1.1 tersangkut di ‘CONNECTING (MENYAMBUNGKAN) atau ‘REASSERTING’ (MENEKANKAN KEMBALI). Apa yang harus saya lakukan?

Ini dapat terjadi jika sambungan jaringan Anda terganggu sementara atau jika koneksi data Anda memblokir sambungan 1.1.1.1:

  • Nonaktifkan aplikasi menggunakan toggle pada halaman beranda aplikasi.
  • Buka aplikasi Pengaturan iOS dan navigasikan ke Pengaturan > Umum > Pengaturan Ulang.
  • Ketuk Atur Ulang Pengaturan Jaringan.
  • Reboot perangkat Anda; pastikan Wifi Anda tersambung.
  • Coba sambungkan aplikasi lagi.

Jika upaya ini tidak membuat sambungan berhasil, nonaktifkan aplikasi 1.1.1.1 dan kirim laporan bug kepada kami.

Saya tidak dapat mengeklik Oke pada dialog instalasi VPN. Apa yang harus saya lakukan?

Jika Anda tidak dapat mengeklik tombol Oke, mungkin ada aplikasi lain di atas dialog. Beberapa aplikasi yang biasanya menyebabkan hal ini adalah Lux Brightness, Night Mode, Twilight, FileManager, dll. Untuk menghindari masalah ini, tutup atau hapus instalasi semua aplikasi yang mungkin menyebabkan hal ini.

Aplikasi tetap saya aktifkan tetapi saya melihatnya mati setelah beberapa saat. Apa yang terjadi?

Ponsel Anda sedang berusaha mengelola baterai dengan menonaktifkan aplikasi. Misalnya, sebagian besar perangkat Huawei dikenal memiliki pengelolaan daya dan memori yang agresif.

Anda dapat memperbaikinya dengan langkah-langkah di bawah ini:

  • Masuk ke ‘Pengaturan’ di ponsel Anda.
  • Pilih ‘Lanjutan’.
  • Kunjungi ‘Pengelola Baterai’.
  • Buka ‘Aplikasi Dilindungi’.
  • Anda dapat memilih “Izinkan aplikasi tetap berjalan setelah layar dimatikan” untuk aplikasi 1.1.1.1.

Mengapa ada notifikasi membandel untuk aplikasi 1.1.1.1?

Jika menggunakan Android 8 atau versi di atasnya, Anda akan melihat notifikasi membandel dari aplikasi 1.1.1.1 yang berbunyi “Kueri DNS Anda privat dan lebih cepat”.

Ini disyaratkan oleh Android OS (8.0+) untuk semua aplikasi dengan aktivitas latar belakang.

Kami mengatur prioritas notifikasi menjadi “Rendah” agar tidak mengganggu notifikasi penting lainnya di ponsel Anda.

Berikut ini satu hal yang dapat Anda lakukan untuk menyembunyikan notifikasi. Harap perhatikan bahwa ini akan membuat Anda tidak lagi menerima notifikasi lainnya dari aplikasi 1.1.1.1. Meskipun begitu, ini dia:

  1. Buka Pengaturan.
  2. Aplikasi & Notifikasi.
  3. Buka pengaturan aplikasi 1.1.1.1 dari daftar aplikasi.
  4. Ketuk “Notifikasi” dan nonaktifkan.

Saya menonaktifkan notifikasi untuk aplikasi 1.1.1.1 tetapi saya sekarang melihat notifikasi lain yang berbunyi “1.1.1.1 sedang menggunakan baterai”. Ada masalah dengan ini? Bagaimana cara menghapusnya?

Anda tidak perlu khawatir. Karena Anda menonaktifkan notifikasi untuk aplikasi tersebut, OS Android sekarang mengirim notifikasi lain yang memperingatkan Anda tentang penggunaan baterai bukan karena aplikasi mengonsumsi banyak baterai tetapi karena aplikasi berjalan di latar belakang (dalam kasus ini, mengamankan semua kueri DNS Anda!).

Berikut ini cara Anda dapat menghapusnya:

  1. Tekan lama notifikasi hingga berubah dan menampilkan toggle.
  2. Ketuk toggle, pastikan warnanya abu-abu.
  3. Pilih “Selesai” untuk menonaktifkan notifikasi.

Saya tidak dapat menambahkan profil VPN. Bagaimana cara memperbaikinya?

Anda mungkin menggunakan VPN lain yang mengaktifkan pengaturan “Selalu Aktif”. Harap nonaktifkan pengaturan tersebut dan aktifkan 1.1.1.1.
