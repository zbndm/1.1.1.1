常见问题

一般问题

什么是 1.1.1.1？

当您请求访问 cloudflare.com 等应用程序时，您的计算机需要知道将您连接到哪个服务器，以便加载该应用程序。计算机不知道如何转换此名称，因此请求专用服务器来为它们执行此操作。

这种专用服务器称为 DNS 递归解析器。解析器的工作是找到给定名称的地址，例如，cloudflare.com 的地址名称为 2400:cb00:2048:1::c629:d7a2，然后将其返回到发送请求的计算机。

计算机被配置为与 IP 地址标识的特定 DNS 解析器进行通信。通常，如果您在家庭或无线 Internet 上，该配置由您的 ISP（例如 Comcast 或 AT&T）管理，如果您连接到了办公室 Internet，则由网络管理员管理。您还可以自行更改与计算机通信的已配置 DNS 解析器。

DNS 解析器有什么作用？

解析器如何知道怎样找到域名的地址？解析器自顶向下进行解析。

每个解析器都知道如何找到域名末尾看不见的 ‘.’（例如 cloudflare.com.）。全球有数百个服务器托管 ‘.’ 文件。通过与 ISC合作，Cloudflare 在其全球所有服务器上均托管该文件。解析器进行了硬编码，知道这些服务器的 IP 地址。

解析器会询问其中一个根服务器，在何处可以找到该链中的下一个链接、TLD 或域结尾。比如 .com 或 .org 等 TLD。幸运的是，这些根服务器存储了所有 TLD 服务器的位置，因此根服务器能够返回 DNS 解析器下一步要询问的 IP 地址。

解析器接下来会询问 TLD 的服务器，在何处能够找到域，例如，解析器可能会向 .com 询问在何处能够找到 cloudflare.com。TLD 托管的文件包含使用此 TLD 的每个域的位置。解析器获得最终 IP 地址后，会将此结果返回给发出询问的计算机。

具有托管这些信息的服务器（称为授权 DNS）和寻求这些信息的服务器（DNS 解析器）组成的整个系统被称为域名系统 (DNS)。DNS 就像用于 Internet 的 Google Maps。它将地点的名称转换为地址，以便您能够弄清楚如何到达目的地。

1.1.1.1 是否支持 IPv6？

是的，1.1.1.1 全面支持 IPv6。

如何清除 1.1.1.1 的 DNS 高速缓存？

您可以通过使用 Purge Cache 工具，刷新用于域名的 1.1.1.1 DNS 高速缓存。您必须输入此域名，选择 DNS 记录类型（或多个类型），然后点击 ‘Purge Cache’ 按钮。

1.1.1.1 在中国能否使用？

在中国无法使用。您可以配置您的 DNS，使其指向 1.1.1.1，但它不会到达我们位于中国的数据中心之一。该请求将路由到我们在中国大陆之外的一个数据中心。

我在哪里能够找到 1.1.1.1 的开发人员文档？

移动应用

此移动应用如何工作？

该应用在您的手机内运行本地解析器，要求所有其他应用将其 DNS 请求发送到此解析器的地址。这些 DNS 请求到达本地解析器后，解析器将其加密，然后发送到 1.1.1.1。

1.1.1.1 应用是一个 VPN 吗？

不是。VPN 应用一般代理您的所有网络流量。1.1.1.1 应用仅替代并保护您手机的 DNS 流量。

DNS 日志是什么？

iOS 上的 DNS 日志包含您手机进行的所有 DNS 查询。您的应用和移动浏览器通常会生成这些 DNS 查询。

控制台日志有什么作用？

控制台日志用于帮助 Cloudflare 调试您的连接。

我能够通过什么方式提出改进建议？

我们会查看您的所有建议。当您使用该应用时，可以摇动手机发送您的改进建议。或者，您可以转到应用菜单，然后点击“帮助”来发送您的反馈。

使用基于 TLS 的 DNS 和基于 HTTPS 的 DNS 有什么区别？

基于 TLS 的 DNS 和基于 HTTPS 的 DNS 都对来自手机的普通 DNS 查询进行加密。

基于 HTTPS 的 DNS 使用端口 443，基于 TLS 的 DNS 使用端口 853。在某些网络中，其中一个端口可能被阻止。如果端口 443 被阻止，您应使用基于 TLS 的 DNS。如果端口 853 被阻止，您应使用基于 HTTPS 的 DNS。在某些情况下，基于 TLS 的 DNS 可能比基于 HTTPS 的 DNS 速度快，而有些情况下却恰好相反。

我能够通过什么方式发送错误报告？

您可以摇动该应用发送错误报告。当您进行该操作时，该应用会在发送错误报告的同时附加屏幕截图和控制台日志。控制台日志可帮助我们调试您的连接，以及改善您的整体应用体验。

当我正在使用 1.1.1.1 应用时，我的 Internet 停止工作。我应该怎么做？

如果发生这种情况，请尝试使用该应用主页中的切换功能关闭该应用，然后再将其打开。如果此问题仍然存在，请禁用该应用，并向我们发送错误报告。

1.1.1.1 应用停滞在“正在连接”或“正在重新声明”状态。我应该怎么做？

如果您的网络连接暂时中断或者您的数据连接正在阻止 1.1.1.1 连接，则可能会发生这种情况：

  • 使用该应用主页上的切换功能禁用该应用。
  • 打开“iOS 设置”应用，导航到“设置”>“常规”>“重置”。
  • 点击“重置网络设置”。
  • 重启您的设备；确保 Wifi 已连接。
  • 再次尝试连接该应用。

如果不允许连接，则禁用 1.1.1.1 应用，并向我们发送错误报告。

我无法在 VPN 安装对话框上单击“确定”。我应该怎么做？

如果您无法单击“确定”按钮，可能是该对话框上面有另一个应用程序。导致此问题的一些已知应用有 Lux Brightness、Night Mode、Twilight、FileManager 等。要避免此问题，请关闭或卸载可能导致此问题的所有应用。

我一直在启用该应用，但过一会儿应用就关闭了。发生了什么情况？

您的手机正在尝试通过禁用该应用来管理电池。例如，大多数华为设备都以激进的内存和电源管理而闻名。

您可以通过执行以下步骤来修复此问题：

  • 转到手机的“设置”。
  • 选择“高级”
  • 访问“电池管理器”
  • 打开“受保护应用”
  • 对于 1.1.1.1 应用，您可以选择“锁屏后允许应用保持运行”。

为什么 1.1.1.1 应用会持续发出通知？

如果您使用的是 Android 8 或更高版本，您将看到来自 1.1.1.1 应用的持续通知，提示“您的 DNS 查询不仅私密而且速度更快”。

对于具有后台活动的所有应用，这是 Android OS (8.0+) 所要求的。

我们将通知的优先级设置为“低”，以便不会干扰您手机上的其他重要通知。

您可以执行以下操作来隐藏通知。请注意，此操作将阻止您接收来自 1.1.1.1 应用的其他通知。但是，您可以：

  1. 转到“设置”。
  2. 应用和通知
  3. 从应用列表中打开 1.1.1.1 应用设置。
  4. 点击“通知”并将其禁用。

我已禁用 1.1.1.1 应用的通知，但现在我看到另一个通知提示“1.1.1.1 正在消耗电池电量”。我应对此感到担心吗？如何将其删除？

您不必担心。由于您已禁用该应用的通知，Android OS 现在会发送另一个通知，提醒您有关电池使用的情况，不是因为该应用消耗了大量电池电量，而是因为该应用正在后台运行（在这种情况下，保护您的所有 DNS 查询！）。

以下是删除方法：

  1. 长按通知，直到更改并显示切换开关。
  2. 点击切换开关，确保其显示为灰色。
  3. 选择“完成”以禁用通知。

无法添加 VPN 配置文件。应该如何解决？

您可能正在使用另一个已启用“始终开启”设置的 VPN。请将其禁用，然后启用 1.1.1.1。

您可以使用以下步骤转到该页面：

  1. 设置
  2. 网络和 Internet
  3. 高级
  4. VPN
  5. 1.1.1.1 的齿轮按钮
  6. 为 1.1.1.1 启用“始终开启”
